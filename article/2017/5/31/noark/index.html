    Noark2是一个游戏服务器端框架，可快速开发出一个易维护、易扩展且稳定高能的游戏服务器，让开发者专注于逻辑功能的开发。精心设计过的它大大简化了网络编程和多线程编程，还有很多改良版的工具类库，如SQL，XML，Excel，Json，AMF3等等。他就是为了解决开发中那些重复劳动而产生的框架。

    我们的目标：稳定、高性能、可扩展、易维护、提高开发效率，我们没有要取代谁，也没有要超越谁，我们只做我们自己。

    Noark2的主要特色：

    • 模块化设计，简洁的API，且无第三方依赖
    • 功能齐全，IOC,log,template应有具有
    • 采用串行化设计理念，优化多线程编程
    • 防外挂，反私服


    快速体验

    使用Gradle

    Noark2使用Gradle管理依赖，在项目中添加对应的依赖即可使用Noark2：

    compile 'xyz.noark:noark-core:1.+'

    第一个服务器

    启服入口

    public static void main(String[] args) {
    Noark.start(new GameServerStartup());
}

    这个类就是一个简单带有Main方法启动入口。

    启服辅助类

    GameServerStartup 用来定制启服配置文件和启服流程.

    public class GameServerStartup extends AbstractServerStartup<GameConfig>{}

    如果不想改启动配置，请参考DefaultNoarkConfig。

    定制GameConfig

    GameConfig是游戏启服配置文件，定制一个GameConfig即可实现自己的特色。以下是GameConfig的代码：

    @XmlFile(name = "game-config.xml")
public class GameConfig extends DefaultNoarkConfig {}

    这里通过XmlFile注解来指定配置文件。
